## COURSE PURPOSE [PRIVATE]

ISM2411: Python for Business is an introductory programming course in the USF Muma College of Business curriculum. It is a required course in the Information Systems major and serves as the gateway to the IS programming sequence — ISM3232 (Business Application Development) is the direct continuation, and ISM2411 is a prerequisite for ISM4314 and most data analytics electives in the college.

The course fills the practical gap between spreadsheet-based data work — which many Muma students already do well — and professional data scripting and analysis. Unlike a traditional computer science introduction, every concept in this course is grounded in business scenarios: pricing formulas, sales transactions, customer records, and retail datasets. Students who complete ISM2411 not only have the prerequisite skills for upper-division IS coursework; they also leave with a GitHub portfolio of 15 labs and a capstone data analysis they can show a recruiter.

This is a core IS course, not an elective. It is appropriate for students who have never written a line of code. Students who already know Python should enroll in ISM3232 or ISM4314 instead.

## COURSE FORMAT [PUBLIC]

ISM2411 is offered in a hybrid format. Each week has two sessions: a lecture session (typically Tuesday) and a lab session (typically Thursday). Lectures introduce and demonstrate new concepts with live coding — students are expected to follow along in VS Code. Labs are structured practice sessions where students work through guided exercises with the instructor available for real-time support.

Every week follows the same four-step rhythm:
1. Complete the assigned reading on the course website before lecture
2. Attend lecture and code along
3. Attend lab and complete the lab assignment
4. Submit the completed lab by Sunday at 11:59 PM

Class time is mostly active: follow-along coding during lecture, independent and peer-supported coding during lab. Eight DataCamp courses run in parallel with lecture content and are completed independently outside class time.

Learning is assessed through weekly labs and quizzes (35%), eight required DataCamp courses (15%), a 75-minute midterm exam (20%), a capstone retail analysis project (25%), and participation (5%).

## STUDENT LEARNING OUTCOMES [PUBLIC]

By the end of this course, students will be able to:

1. Explain how a computer stores and executes a program; navigate the file system using the command line
2. Write Python scripts that use variables, conditionals, loops, and functions to automate repetitive business calculations
3. Read, write, and manipulate structured data using lists, dictionaries, and the csv module
4. Load datasets from CSV files using pandas, clean messy data, and compute descriptive statistics
5. Group, filter, and aggregate data with pandas to answer specific business questions
6. Create bar, line, and histogram charts with matplotlib and seaborn; choose the appropriate chart type for the data
7. Use Git and GitHub to version-control, organize, and submit code professionally
8. Use AI coding assistants responsibly — verify outputs, debug errors independently, and disclose AI-generated code per course policy

## COURSE OBJECTIVES [PUBLIC]

Students will work toward the learning outcomes above through the following course objectives:

1. Navigate the command line and file system to set up, run, and organize Python projects
2. Write Python scripts using variables, operators, conditionals, loops, and functions to solve business calculation problems
3. Use lists, dictionaries, and the csv module to store and retrieve structured business data
4. Load, clean, summarize, and visualize business datasets using the pandas and matplotlib libraries
5. Apply version control with Git and GitHub to organize, track, and submit code professionally
6. Evaluate AI coding tools critically: understand when to use them, how to verify their output, and how to disclose their use appropriately

## HOW TO SUCCEED IN THIS COURSE [PUBLIC]

This course moves quickly, and each module builds on the previous one. Here is what works:

**Do the reading before lecture.** Lecture time reinforces and extends the reading — it does not replace it. Students who skip readings arrive two steps behind.

**Attend every lab.** Labs are structured practice that would take two to three times longer to complete alone. Missing lab hurts more than missing lecture.

**Use the self-check quizzes as your first study session.** Questions marked ★ MIDTERM-ELIGIBLE or ★ CAPSTONE-ELIGIBLE appear on formal assessments verbatim or with minor variation. Treat the quiz as early exam prep.

**Run your code often.** Do not write 20 lines and then run. Write 3 lines, run, check, continue. Errors are easier to find when you introduce them one at a time.

**Read the full traceback.** The answer to most errors is in the last two lines of the traceback. Read what Python is telling you before searching online.

**Come to office hours with a specific error.** Not "I'm confused." Bring your code, the error message, and what you tried. You will leave office hours understanding something you did not understand before.

**What you should already know:** This course assumes no prior programming experience and no statistics background. If you have used Excel, Sheets, or any no-code data tools, that background will help. If you want to get ahead, complete the pre-course DataCamp track recommended on the course website before Module 1.

## COURSE POLICIES: GRADES [PUBLIC]

### First Day Attendance Policy
Students must complete the **Module 1 Syllabus Quiz** on Canvas by the end of the first week of class. This quiz confirms your enrollment and active participation in the course. Students who do not complete the quiz by the deadline are subject to being dropped from the course per USF policy.

### Late Work Policy
Weekly labs are due **Sunday at 11:59 PM** of the module week they are assigned. Late submissions are penalized **10% per calendar day** for up to three days. After three days, the lab receives a zero and will not be accepted.

Each student has **one 48-hour no-questions-asked extension** per semester. To use it, email the instructor before the deadline with the subject line "ISM2411 Extension — Module [number]." Extensions cannot be used for the midterm exam or the capstone project.

### Make-up Exams Policy
If you cannot attend the midterm exam due to a verified illness or documented emergency, email the instructor before the exam begins. Makeup exams are arranged at the instructor's convenience and require documentation (e.g., a verification-of-care letter from Student Health Services). Unannounced absences from the midterm receive a zero — there is no default makeup window.

### Extra Credit Policy
The only extra credit available is completion of the two optional DataCamp courses, worth up to +5% added to the DataCamp component. No other extra credit will be offered under any circumstances.

### Rewrite Policy
No rewrites or resubmissions of labs, quizzes, or the capstone are permitted. Invest in getting it right the first time.

### Exam Retention Policy
Midterm exams are reviewed in class on the day they are returned, then collected. Exams are retained for one semester following the current semester and then destroyed. Students who wish to review their exam may do so during office hours within the one-week grade dispute window.

### Group Work Policy
All work in this course is individual. Discussion of concepts and general debugging approaches with classmates is permitted and encouraged. Sharing code — whether by showing your screen, sending a file, or posting in a chat — is academic dishonesty. Each student's submitted code must be written independently. If you debug together, each person must write their own solution from scratch afterward. The standard: can you explain every line of your submission without help? If not, it is not yours.

### Final Examinations Policy
All final examinations are scheduled in accordance with USF's Final Examination Policy. The capstone presentation in Module 16 counts as this course's final assessment. Students must be available for the scheduled presentation window.

## GENERATIVE AI POLICY [PRIVATE]

This course teaches professional, disclosed use of AI coding tools — not avoidance of them. Three approved modes are defined:

**Permitted uses:**
- **Explainer mode:** Ask AI to explain a concept or a piece of code you don't understand. Always encouraged.
- **Debugger mode:** Paste an error message and ask AI to explain what went wrong. Allowed — but try to read the traceback yourself first.
- **Generator mode:** Ask AI to write code for you. Allowed *only if* you (a) understand every line it produces, (b) can explain it without the AI, and (c) disclose that AI generated it in your submission comment using the exact disclosure language on the AI Policy page.

**Prohibited uses:**
- Using AI to complete the midterm exam in any form — this is academic dishonesty.
- Submitting AI-generated code without disclosure.
- Using AI to write code you cannot explain independently.

See the full AI Use Policy page on the course website for examples of each mode and the required disclosure language. See also: USF Provost guidance at https://www.usf.edu/provost/faculty-success/teaching-learning/guidance-generative-ai.aspx.
